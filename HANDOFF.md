## What This Project Does

Aggregates 2026 US election data (Senate, House, Governor) from polling and prediction-market sources into a unified dataset, then powers a dashboard at `docs/index.html` (deployed to GitHub Pages from `/docs`).

Three tabs:
1. **Arb Scanner** — cross-platform price mismatches, with stake sizing and now per-side tradeable depth
2. **Poll Explorer** — aggregated polling per race
3. **Raw Polls** — every individual poll, sortable by recency

## Arb Scanner Details

### General-election matching
Joins Kalshi/PredictIt/Polymarket on canonical `race_id` (e.g. `2026-SEN-PA`, `2026-H-CA-13`). For each pair, computes:
- `raw_gap_pp` = abs difference in implied Dem-win probability
- `net_gap_pp` = raw_gap minus both fees
- `arb_type`: `guaranteed` if buying Dem on cheap side + Rep on expensive side locks in profit after fees; else `one-sided`
- Stake sizing for guaranteed arbs (in `make_pair` / `compute_arb_math`)

### Primary-candidate matching (added 2026-04-22)
Independent path that matches on `(state, office, district, party, candidate_last, candidate_first)` parsed from market titles — doesn't need a `race_id` to exist. Catches things like "Will Zach Wahls be the Democratic nominee for Senate in Iowa?" priced differently across platforms.

Disambiguation key includes first initial because Chris Sununu vs John E. Sununu collapsed without it (caused a fake 88pp arb).

### Depth feature (added 2026-04-23)
`scripts/fetch_depth.py` queries:
- Kalshi: `GET /trade-api/v2/markets/{ticker}/orderbook?depth=50`
- Polymarket: `GET https://clob.polymarket.com/book?token_id={tid}`

For each side, computes `best_bid`, `best_ask`, `best_bid_size`, `best_ask_size`, `depth_bid_at_1pp`, `depth_ask_at_1pp`, `max_buy_size_at_3pp_edge`. The scanner joins these onto each pair as `depth_a_*` / `depth_b_*`. Dashboard's "Tradeable" column shows the per-side max-at-3pp.

**Why this matters:** the top-by-net-gap pair is often illusory. SEN-VA showed an 82pp net gap but Kalshi's best ask was $0.001 with no real depth — the apparent arb couldn't actually be filled. The new column makes that visible.

## Common Gotchas

- **Polymarket token IDs are 78-digit ints.** Always read with `dtype={"yes_token_id": str, "no_token_id": str}` — pandas will silently corrupt them to floats otherwise. Same for `depth_targets.csv` and `orderbook_depth.csv`: use `dtype={"market_id": str}`.
- **Kalshi v2 prices are decimals** (`yes_bid_dollars` = "0.42") not cents. v1 was 0–100 — don't mix.
- **PredictIt contract sums often exceed 1.0.** A "guaranteed" arb that includes PredictIt may be an artifact of independent Yes/No bets, not a real arb.
- **Same-surname candidates** (Sununu, Kennedy, Bush) need first-initial disambiguation in primary matching. Already handled in `_first_initial()`.
- **GitHub Pages deploys from /docs on push to master.** No build step.
